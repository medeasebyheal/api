# Medease API — Admin endpoints\n+
## Revoke User Access\n+\n+Endpoint: `PATCH /api/users/:id/revoke`\n+\n+Description:\n+- Admin-only endpoint to immediately revoke a user's access.\n+- Actions performed:\n+  - Invalidate all active sessions for the user (forces logout)\n+  - Expire all active user packages (sets status to `expired` and updates `expiresAt`)\n+  - Clears `user.activePlanId`\n+\n+Authentication & Authorization:\n+- Protected by the existing `auth` middleware and `requireRole('admin')`.\n+\n+Response:\n+```\n+{\n+  \"message\": \"Access revoked\",\n+  \"user\": { /* user object without password */ },\n+  \"expiredPackagesCount\": 1,\n+  \"sessionsInvalidatedCount\": 1\n+}\n+```\n+\n+Notes:\n+- No database schema changes required.\n+- For audit logging or temporary suspensions, consider adding an audit collection or expanding `UserPackage.status`.\n+\n*** End Patch
